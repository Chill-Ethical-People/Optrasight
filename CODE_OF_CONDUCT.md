# Code of Conduct

## Our Standard

OptraSight welcomes constructive participation from security engineers,
analysts, researchers, and builders. Keep discussions professional, evidence-led,
and focused on improving the project.

Examples of encouraged behavior:

- Use clear, respectful language.
- Assume good intent while still reviewing code and security claims rigorously.
- Share reproducible details for bugs, tests, and security findings.
- Respect privacy, confidentiality, and responsible disclosure boundaries.

Examples of unacceptable behavior:

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private credentials, client data, exploit payloads, or sensitive
  logs in public issues.
- Encouraging unauthorized access, disruption, or misuse of the platform.
- Repeatedly derailing technical discussions after maintainers ask to refocus.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other
contributions that violate this code of conduct. Maintainers may also restrict
participation for repeated or severe violations.

Security vulnerabilities must be reported privately as described in
[SECURITY.md](./SECURITY.md).
